## 数据来源：43个JD是怎么来的

### 采集方法

这份分析基于43个AI相关岗位的JD（Job Description），数据来源包括：

- **BOSS直聘**：25个岗位，覆盖汕头、东莞、深圳
- **猎聘**：10个岗位，覆盖深圳、佛山
- **实习僧**：8个岗位，覆盖深圳、东莞

### 岗位分布

| 城市 | 岗位数量 | 平均薪资（K/月） |
|------|----------|------------------|
| 深圳 | 18 | 15-25 |
| 东莞 | 12 | 10-18 |
| 汕头 | 8 | 8-15 |
| 佛山 | 5 | 12-20 |

筛选标准：岗位标题包含"AI""人工智能""机器学习""计算机视觉""深度学习"等关键词，排除纯销售和纯管理岗。

## 三条路线能力矩阵

分析完43个JD后，我发现AI岗位的需求可以归纳为三条主要路线。每条路线对技能的要求差异很大，薪资水平也不同。

### 路线一：AI Agent开发（15-25K）

这是近两年新兴的方向，随着大模型应用落地而爆发。

| 技能项 | 要求程度 | 说明 |
|--------|----------|------|
| Python | 必须 | 基础中的基础 |
| LangChain / LlamaIndex | 强烈要求 | Agent框架，几乎所有岗位都提到 |
| RAG（检索增强生成） | 强烈要求 | 企业知识库是刚需 |
| 多Agent协作 | 加分项 | 有经验者优先 |
| Prompt Engineering | 必须 | 核心能力，但JD中很少单独列出 |
| API对接 | 必须 | 需要对接各种第三方API |
| 向量数据库 | 要求 | Milvus、Pinecone、ChromaDB |
| 前端基础 | 加分项 | 简单的Web界面开发能力 |

**典型岗位描述**："负责基于大模型的智能体开发，包括知识库构建、对话系统设计、多轮交互优化。"

### 路线二：工业AI工程师（35-50K）

这是薪资最高的方向，门槛也最高。需要同时具备AI能力和工业领域知识。

| 技能项 | 要求程度 | 说明 |
|--------|----------|------|
| Python | 必须 | 数据处理和模型训练 |
| 预测性维护 | 强烈要求 | 核心业务场景 |
| 机器视觉 | 要求 | 缺陷检测、尺寸测量 |
| 数据采集 | 要求 | 了解PLC、Modbus、OPC UA |
| 时序分析 | 要求 | 传感器数据处理 |
| 边缘计算 | 加分项 | 在产线端部署模型 |
| 项目管理 | 加分项 | 需要独立推进项目 |

**典型岗位描述**："负责工厂智能质检系统和设备健康管理系统的研发，包括数据采集、模型训练、产线部署全流程。"

### 路线三：机器视觉工程师（10-18K）

这是最传统也最成熟的方向，岗位数量最多。

| 技能项 | 要求程度 | 说明 |
|--------|----------|------|
| Python/C++ | 必须 | 至少精通一门 |
| OpenCV | 必须 | 基础视觉库 |
| YOLO系列 | 强烈要求 | 目标检测的标配 |
| Halcon | 要求 | 工业视觉领域的商业软件 |
| 工业相机 | 了解 | 海康、大华等品牌的选型和使用 |
| 光源选型 | 了解 | 不同场景下的打光方案 |
| CAD图纸 | 加分项 | 能看懂机械图纸 |

**典型岗位描述**："负责产品外观缺陷检测系统的开发，使用YOLO/Halcon进行视觉算法设计和调试。"

### 还缺少的技能

| 技能 | 重要程度 | 补齐难度 | 建议学习时间 |
|------|----------|----------|-------------|
| SQL | 高 | 低 | 2周 |
| Docker（深度使用） | 高 | 中 | 3周 |
| PyTorch | 高 | 中 | 4周 |
| PLC/工业协议 | 中 | 高 | 6周 |
| Halcon | 中 | 中 | 3周 |

**SQL**是投入产出比最高的补齐项。很多岗位要求SQL，但学习曲线平缓，两周就能达到面试要求。

**Docker**是部署环节的必备技能。我之前只用过基础的Docker命令，但对于容器编排、Docker Compose多服务部署还需要深入学习。

**PyTorch**是模型训练的核心工具。虽然我现在更多用传统机器学习，但深度学习框架的掌握几乎是必须的。

## 给同届同学的建议

### 不要海投，精准匹配

海投100个岗位不如精准投10个岗位。每个岗位的JD都有明确的技能要求，如果你的项目经验和技能匹配度低于60%，大概率会在简历筛选阶段被淘汰。

建议的做法是：先把JD中的技能要求拆解出来，和自己的能力做对比。匹配度高的岗位重点准备，匹配度低的就跳过。

### 项目经验比刷题重要

对于AI岗位，面试官更看重你做过什么项目，而不是你刷了多少LeetCode题。一个完整的、有数据支撑的项目，比刷100道算法题更有说服力。

当然，算法题也不能完全不准备。建议把常见题型（排序、搜索、动态规划的基础题）过一遍即可，不需要追求hard题的满分。

### 差异化竞争

43个JD中，Python几乎是必备技能。那凭什么选你而不是别人？答案是**差异化**。

如果你只会Python+PyTorch，那你的竞争对手可能是成千上万的应届生。但如果你能同时展示嵌入式开发（ESP32）、机器人系统（ROS2）、或者工业协议（Modbus）的经验，你就进入了另一个竞争维度。

### 薪资预期要合理

不同路线的薪资差异很大：

- 机器视觉工程师：10-18K（入门门槛低，竞争激烈）
- AI Agent开发：15-25K（新兴方向，需求增长快）
- 工业AI工程师：35-50K（门槛最高，但人才稀缺）

不要只盯着高薪岗位。先找到一个能入行、能积累经验的岗位，在工作中持续提升自己，比一步到位更重要。

### 持续学习的优先级

根据差距分析，我的学习优先级是：

1. **SQL**（2周，投入产出比最高）
2. **Docker深度使用**（3周，部署必备）
3. **PyTorch**（4周，模型训练核心）
4. **工业协议**（长期，走工业AI路线必学）

学习不是目的，能用学到的东西做出项目、解决实际问题才是目的。每学一个新技能，最好能对应做一个小项目，形成"学习-实践-输出"的闭环。

但即使是这样一次粗略的分析，也让我对自己的职业方向有了更清晰的认知。与其在焦虑中海投简历，不如花半天时间做一次系统的技能盘点，找到自己的优势和短板，然后有针对性地补强。
