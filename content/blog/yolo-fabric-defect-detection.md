## 需求背景

### 祥发网厂的痛点

汕头祥发网厂是一家专业生产各类网布、面料的工厂。在一次企业调研中，他们向我们描述了几个核心痛点：

**人工质检速度慢。** 一条产线上的布面宽度通常是1.5-3米，质检工人需要逐段检查布面质量。按照每分钟10米的产线速度，一个工人的目检速度很难跟上。

**漏检率高。** 人工质检的漏检率在 5%-15% 之间，具体取决于工人的经验和疲劳程度。尤其是微小的破洞和轻微的色差，人眼很容易忽略。

**人力成本上升。** 一个熟练的质检工人月薪在 6000-8000 元，而且招工越来越难。年轻人不愿意做这种重复性高、眼睛容易疲劳的工作。

```text
工厂现状：
┌──────────────────────────────────────┐
│  产线速度：10m/min                    │
│  人工质检速度：8m/min（跟不上）       │
│  漏检率：5%-15%                       │
│  质检工位：2人/班 × 3班/天            │
│  月度人力成本：约 4.2万               │
│  客户投诉率：月均 3-5次（因质量问题）  │
└──────────────────────────────────────┘
```

他们的需求很明确：**用机器视觉替代人工质检，至少能覆盖80%的常见缺陷类型。**

## 缺陷类型

经过对工厂产品的分析，我们将布面缺陷归纳为四类：

### 破洞（Hole）

```text
特征分析：
- 形状：不规则圆形或椭圆形
- 尺寸：最小 2mm，最大可达 20mm+
- 纹理：与正常布面纹理明显不同
- 检测难度：★★☆☆☆（相对容易检测）
```

破洞是最常见的缺陷类型，特征比较明显。在图像中通常表现为亮度异常区域，与周围正常布面的灰度值差异较大。

### 污渍（Stain）

```text
特征分析：
- 形状：不规则，边界模糊
- 颜色：与布面底色不同（油渍偏黄、水渍偏暗）
- 尺寸：变化范围大
- 检测难度：★★★☆☆（颜色相近时难检测）
```

污渍的检测难点在于**颜色对比度低**的情况。比如浅色布面上的轻微水渍，肉眼都需要仔细看才能发现。

### 断经（Broken Warp）

```text
特征分析：
- 形状：沿经向的线状缺陷
- 宽度：通常只有1-2根纱线
- 长度：从几厘米到几十厘米不等
- 检测难度：★★★★☆（特征细微）
```

断经是四类缺陷中最难检测的。因为它只是缺少了1-2根经纱，在纹理上表现为非常细微的变化。

### 色差（Color Difference）

```text
特征分析：
- 形状：通常为大面积渐变
- 颜色：与标准色存在偏差
- 特征：边界不明显，渐变过渡
- 检测难度：★★★★★（最困难）
```

色差检测的难度在于**缺乏明确的边界**。它不是一个"点"或"线"的缺陷，而是一个"面"的问题。而且色差的判定标准本身就很主观——什么样的偏差算"色差"？需要和客户协商确定阈值。

## 数据采集与标注

### 工厂环境获取训练数据的挑战

在工厂环境下采集训练数据，远比在实验室里困难：

**光照不一致。** 工厂的照明条件随时间变化——白天和晚上的光照不同，不同工位的照明条件也不同。同一个缺陷在不同光照下的表现可能差异很大。

**布面种类多。** 不同颜色、不同纹理、不同材质的布面，缺陷的表现形式都不一样。用一种布面训练的模型，换到另一种布面上效果可能大打折扣。

**正样本稀缺。** 缺陷在正常生产中出现的频率不高，需要大量采集才能积累足够的正样本。

我们的采集方案：

```text
硬件：
- 工业相机：海康MV-CS系列，分辨率2048×1024
- 光源：LED条形光源，可调节角度和亮度
- 安装位置：产线质检工位正上方

采集策略：
- 连续采集：产线运行期间不间断抓拍
- 手动触发：发现缺陷时人工触发保存
- 增强采集：在已知缺陷位置附近加密采集
```

### 标注策略

我们使用 LabelImg 进行标注，采用 YOLO 格式：

```text
标注规范：
├── 图片命名：{布料类型}_{日期}_{序号}.jpg
├── 标注格式：YOLO txt（class x_center y_center width height）
├── 标注比例：训练集80% / 验证集15% / 测试集5%
└── 标注质量控制：双人标注 + 交叉验证
```

最终数据集规模：

| 缺陷类型 | 样本数 | 占比 |
|---------|--------|------|
| 破洞 | 1,247 | 35% |
| 污渍 | 1,063 | 30% |
| 断经 | 892 | 25% |
| 色差 | 354 | 10% |
| **总计** | **3,556** | **100%** |

色差样本最少，这也是色差检测效果相对较差的原因之一。

## YOLOv8模型训练

### 模型选型 trade-off

YOLOv8 提供了多个规模的模型：n（nano）、s（small）、m（medium）、l（large）、x（xlarge）。在工业部署场景下，需要在**速度和精度之间做权衡**：

```text
模型对比（在我们的数据集上测试）：
┌──────────┬──────────┬──────────┬──────────────┐
│ 模型      │ mAP@0.5  │ 推理速度  │ 模型大小      │
├──────────┼──────────┼──────────┼──────────────┤
│ YOLOv8n  │ 0.78     │ 3ms/张   │ 6.2MB        │
│ YOLOv8s  │ 0.85     │ 5ms/张   │ 22.5MB       │
│ YOLOv8m  │ 0.89     │ 12ms/张  │ 52.0MB       │
│ YOLOv8l  │ 0.91     │ 25ms/张  │ 87.3MB       │
└──────────┴──────────┴──────────┴──────────────┘
```

产线速度是 10m/min，布面宽度 1.5m，每帧覆盖区域 1.5m × 0.3m（按30%重叠），需要的帧率大约是 1fps。所以即使是 YOLOv8n 也完全满足实时性要求。

最终选择了 **YOLOv8s**，在精度和速度之间取得了较好的平衡。

### 训练代码

```python
from ultralytics import YOLO

model = YOLO('yolov8s.pt')  # 选择small模型平衡速度与精度
results = model.train(
    data='fabric_defect.yaml',
    epochs=100,
    imgsz=640,
    batch=16,
    augment=True,
    mosaic=1.0,
    mixup=0.2
)
```

### 数据增强策略

针对工厂环境的数据特点，我们重点使用了以下增强策略：

```text
增强策略：
├── Mosaic（mosaic=1.0）：将4张图拼接，增加小目标检测能力
├── Mixup（mixup=0.2）：图像混合，提高模型泛化能力
├── 亮度调节：模拟不同光照条件
├── 高斯噪声：模拟工业相机的噪声
└── 随机裁剪：模拟不同拍摄距离
```

训练100个 epoch 后，模型在验证集上的表现：

```text
训练结果：
├── mAP@0.5: 0.85
├── mAP@0.5:0.95: 0.62
├── Precision: 0.88
├── Recall: 0.82
└── 最佳 epoch: 87
```

## LLM报告生成

检测结果如果只是输出一个 bounding box，对工厂管理人员来说不够友好。我们集成了 MiniMax LLM，将检测结果转化为**结构化的缺陷报告**。

### 报告生成流程

```python
def generate_defect_report(detections: list) -> str:
    """将检测结果转化为结构化报告"""
    
    # 统计各类缺陷数量
    defect_summary = {}
    for det in detections:
        defect_type = det['class_name']
        if defect_type not in defect_summary:
            defect_summary[defect_type] = 0
        defect_summary[defect_type] += 1
    
    # 构建 Prompt
    prompt = f"""
    根据以下布面缺陷检测结果，生成一份质检报告：
    
    检测时间：{datetime.now().strftime('%Y-%m-%d %H:%M')}
    布料类型：{fabric_type}
    检测长度：{inspection_length}米
    
    缺陷统计：
    {json.dumps(defect_summary, ensure_ascii=False)}
    
    请生成包含以下内容的报告：
    1. 缺陷总览
    2. 各类缺陷详情及可能原因
    3. 改进建议
    """
    
    report = llm.generate(prompt)
    return report
```

## 工厂部署挑战

从实验室到工厂，中间隔着"200米"的距离。这200米里充满了实际问题。

### 光照不一致

工厂的光照条件是最大的挑战。我们的解决方案是**硬件+软件双管齐下**：

```text
硬件层面：
- 使用恒定亮度的LED条形光源
- 安装遮光罩，减少环境光干扰
- 相机参数固定曝光时间

软件层面：
- 图像预处理：直方图均衡化
- 训练时加入大量光照变化的增强
- 考虑使用自适应阈值
```

### 布面纹理干扰

布面的编织纹理在图像中会形成周期性的明暗变化，这对缺陷检测造成干扰。尤其是**断经**这种缺陷，和纹理变化非常相似。

解决方案是**预处理去除周期性纹理**：

```python
import cv2
import numpy as np

def remove_texture_pattern(image):
    """通过频域滤波去除布面周期性纹理"""
    # 傅里叶变换
    f = np.fft.fft2(image)
    fshift = np.fft.fftshift(f)
    
    # 创建带阻滤波器，去除纹理对应的频率
    rows, cols = image.shape
    crow, ccol = rows // 2, cols // 2
    mask = np.ones((rows, cols), np.uint8)
    
    # 布面纹理通常在特定频率上有峰值
    for freq in texture_frequencies:
        cv2.circle(mask, (ccol + freq, crow), 5, 0, -1)
        cv2.circle(mask, (ccol - freq, crow), 5, 0, -1)
    
    # 应用滤波器
    fshift = fshift * mask
    f_ishift = np.fft.ifftshift(fshift)
    img_back = np.fft.ifft2(f_ishift)
    
    return np.abs(img_back)
```

### 实时性要求

产线速度 10m/min，要求系统至少每秒处理一帧图像。在 GPU 上，YOLOv8s 的推理时间约 5ms/帧，完全满足要求。但需要考虑**前后处理的时间**：

```text
单帧处理时间分解：
├── 图像采集：50ms
├── 预处理：10ms
├── YOLO推理：5ms（GPU）
├── 后处理：5ms
├── 报告生成：异步（不影响实时性）
└── 总计：70ms/帧（远低于1000ms的实时性要求）
```

## 效果与反思

### 实际效果

经过在祥发网厂的试运行，系统达到了以下效果：

```text
系统性能指标：
├── 检测准确率：85.3%
├── 漏检率：3.2%（从人工的5-15%降低到3.2%）
├── 误检率：8.5%（可接受范围）
├── 单帧处理时间：70ms
└── 日均检测量：约5000米
```

### 反思

**数据质量比模型选择更重要。** 一开始我们花了很多时间调优模型参数，后来发现真正限制效果的是训练数据的质量。标注不一致、样本不均衡、场景覆盖不全——这些问题的优先级远高于换一个更大的模型。

**工业落地不是技术问题，是工程问题。** 实验室里 90% 的 mAP，到了工厂可能只有 70%。光照、振动、温度、灰尘——这些现实因素对系统的影响比你想象的大得多。

**LLM 报告生成是锦上添花。** 工厂管理人员反馈，他们最看重的是**检测的准确性和实时性**，报告生成是加分项但不是必需品。

```text
优先级排序（工厂视角）：
1. 不漏检（Recall）  ← 最重要
2. 实时性（Speed）   ← 其次
3. 不误检（Precision）← 再次
4. 报告美观（Report）← 加分项
```

这个项目让我深刻理解了一句话：**从实验室到工厂的距离，不是物理距离，而是工程思维的距离。** 技术只是起点，真正的挑战在于如何让技术在真实环境中稳定工作。
